📄 Documentation de l'API - Port de Plaisance de Russell

🔐 Authentification

  • POST /api/auth/register – Créer un compte (bibliothécaire)
  • POST /api/auth/login – Se connecter (retourne un token JWT)

👤 Utilisateurs

⚠️ Requiert un token dans l'en-tête Authorization

  • GET /api/users – Liste des utilisateurs
  • GET /api/users/:id – Détail d’un utilisateur
  • PATCH /api/users/:id – Modifier un utilisateur
  • DELETE /api/users/:id – Supprimer un utilisateur

🛶 Catways

⚠️ Requiert un token

  • GET /api/catways – Liste des catways
  • GET /api/catways/:id – Détails d’un catway
  • POST /api/catways – Créer un catway
  • PATCH /api/catways/:id – Modifier un catway
  • PUT /api/catways/:id – Remplacer un catway
  • DELETE /api/catways/:id – Supprimer un catway

📅 Réservations

⚠️ Requiert un token

  • GET /api/reservations – Liste de toutes les réservations
  • GET /api/catways/:id/reservations – Réservations d’un catway
  • GET /api/catways/:id/reservations/:idReservation – Détail d’une réservation
  • POST /api/catways/:id/reservations – Créer une réservation
  • DELETE /api/catways/:id/reservations/:idReservation – Supprimer une réservation

🔐 Sécurité

  • Toutes les routes protégées doivent inclure dans l'en-tête :
  • Authorization: Bearer <votre_token>
  • Le token JWT est généré à la connexion et stocké dans le navigateur (ex: localStorage).
